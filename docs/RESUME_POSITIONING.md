# Resume Positioning

## Project title

**Transaction Risk Engine**

## One-line resume description

Built a production-style transaction fraud risk engine using leakage-safe behavioral feature engineering, LightGBM/XGBoost, imbalance-aware evaluation, SHAP explainability, FastAPI inference, and a Streamlit monitoring dashboard.

## ML-focused bullet

Engineered point-in-time transaction, frequency, and behavioral aggregation features from IEEE-CIS fraud data and trained LightGBM/XGBoost models evaluated using PR-AUC, ROC-AUC, Recall@Top-K, and cost-sensitive thresholds.

## SDE-focused bullet

Developed a modular ML inference system with FastAPI endpoints, serialized model artifacts, reproducible training scripts, unit tests, Docker-ready structure, and a Streamlit dashboard for transaction risk exploration.

## Data science-focused bullet

Designed an imbalanced fraud detection workflow with time-based validation, feature ablations, precision-recall analysis, threshold selection, and SHAP-based transaction-level explanations.

## Advanced bullet after extensions

Added entity graph risk features, drift monitoring using PSI/KS tests, and champion-challenger model comparison to simulate production ML monitoring for changing fraud patterns.

## Interview explanation

Credit default predicts whether a customer will repay later. Transaction Risk Engine scores a specific transaction happening now and decides whether to approve, review, or block it based on behavior and entity relationships.
